# Presentation Zen — Using Visual Aids in Presentations

## TL;DR
Visual aids exist to **enhance, not replace**, your spoken message. Apply Garr Reynolds' *Presentation Zen*: simplicity, restraint, naturalness. The classic failure mode is the text-heavy slide that gets read aloud — the audience reads instead of listens, and retention collapses.

## Why visuals matter
- **Memory retention: ~65% with a relevant visual vs ~10% without.** See [[visual-memory-retention]].
- Visuals deliver **clarity** (simplify complex info — e.g. flowchart vs wall of text) and **engagement** (hold attention).

## Five core principles ([[presentation-zen]])
1. **Keep it simple** — one main idea per slide; spread depth across multiple slides.
2. **Use high-quality images** — relevant, unique; avoid cliché stock.
3. **Minimize text** — apply the [[six-by-six-rule]]; prefer charts/icons/diagrams to words.
4. **Use contrast** to highlight what matters (e.g. colour the key trend, mute the rest).
5. **Storytelling** — slides as chapters; emotional and logical narrative connects the deck.

## Visual aid types — when to use what

| Type                | Best for                          | Best practice                                          |
|---------------------|-----------------------------------|--------------------------------------------------------|
| Graphs / charts     | Data, trends, comparisons         | Clean; label axes; 1–2 lines per chart; annotate trends|
| Diagrams / flowcharts | Processes, relationships        | One concept per diagram; avoid branch explosion        |
| Images / photos     | Emotional impact, real context    | High-res; directly relevant; not abstract              |
| Icons / symbols     | Quick concept representation      | Recognisable; consistent style; don't overuse          |

## Practical tips
- **Limit animation** — purposeful (e.g. bullet reveal), not flashy.
- **Maintain consistency** — same palette, typeface, layout throughout.
- **Leave white space** — don't fill every pixel; "less is more".

## Common pitfalls (likely exam fodder)
- **Overloading slides** — too many elements per slide
- **Ignoring accessibility** — low contrast, tiny text. Use browser/automated checkers.
- **Relying too heavily on visuals** — they support narration, never replace it. Never read off the slide.
